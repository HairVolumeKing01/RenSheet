# RenSheet

二次元拼团金额计算工具集。纯前端，浏览器内完成 Excel 解析、计算、导出。

## 技术栈

| 层 | 技术 |
|---|---|
| 页面 | HTML5 / CSS3 / Vanilla JS |
| Excel 解析 | [SheetJS](https://sheetjs.com/) (CDN) |
| Excel 导出 | [ExcelJS](https://github.com/exceljs/exceljs) (CDN) |
| 持久化 | localStorage 自动保存草稿 |
| 部署 | Cloudflare Pages / 任意静态托管 |

无框架、无构建工具、无后端依赖。所有计算在用户浏览器完成，文件不会上传到任何服务器。


## 页面功能

| 页面 | 入口 | 功能 |
|---|---|---|
| 肾表 | shenbiao.html | 导入排表 Excel → 按人聚合 → 分类式/清单式肾表导出。支持多 sheet 单文件或分表上传两种模式 |
| 国际表 | guoji.html | 排表 + 重量参数 → 按重量分摊国际运费。支持多 sheet 单文件或分表上传两种模式 |
| 二调退补表 | RatioFix.html | 原排表 vs 拆配排表 → 按角色热度自动配平或手动配平 → 每人补款/退款明细。仅支持单 sheet 排表 |
| 打赏 | dashang.html | 老板请赏光，黑金 KTV 主题赞赏页 |

## 数据流

```
导入排表(.xlsx) → SheetJS 解析 → 按人聚合 → 生成表格 → 预览 → ExcelJS 导出
```

### 排表格式规范

每个排表文件可包含多个品类（每个 sheet 一个品类），也可分文件上传：

| 行 | 内容 | 说明 |
|---|---|---|
| A1 | 品类名 | 如"吧唧"、"立牌"，不可为空 |
| B1 起 | 必须为空 | 格式校验项 |
| 第1行 | 角色名列头 | 保持原文，不缩写不删字 |
| 第2行 | 单价 | 该角色谷子单价 |
| 第3行起 | 购买数据 | 每格填购买人名字，不买留空 |

## 核心函数

### 解析层
- `getSheetKind(wb)` — 校验排表格式，提取 A1 品类名
- `parseSheet(wb)` — 遍历数据行，生成 `{person, character, label, kind, price}` 购买记录
- `colToLetter(col)` — 列号转字母（0→A），用于错误定位
- `tryParsePreview(wb)` — 导入时即解析，返回 CN/角色/记录统计

### 聚合层
- `processAllFiles()` — 合并所有文件，按人名聚合到 `personMap`
- `getSortedKinds(personMap)` — 品类排序（吧唧>透卡>立牌>其他）

### 输出层
- `buildCategorizedTable(data)` — 分类式表格
- `buildListTable(data)` — 清单式表格
- `renderPreview()` — HTML 预览渲染
- ExcelJS 导出 — 主题色标题行/表头/交替数据行

### 草稿系统
- `saveDraft()` — 序列化状态到 localStorage
- `restoreDraft()` — 页面加载时自动恢复
- `clearDraft()` — 导出成功后清除

## 主题系统

16 套颜色主题：原有 12 套冷色与中性色主题，以及琥珀、陶土、暖玫瑰、暖棕 4 套暖色主题，统一应用于表格标题行、表头、交替数据行和 Excel 导出。

## 肾表定尾模式

- 固定定金直接输入金额，例如 `20`；比例定金必须带 `%`，例如 `30%`。
- 尾款 = 总肾 − 定金。比例模式下，定金 = 总肾 × 比例。
- 开启后输出金额列为“定金 | 尾款 | 总肾”，并校验定金合计 + 尾款合计 = 总肾合计。

## 二调退补表当前规则

- 调价器以拆配表角色为准，原表独有角色按拆配点数 0 处理且不显示。
- 原调价 = 原排表单价 − 原排表均价；二调从原调价开始，不读取拆配表第三行作为调价起点。
- 上传两张排表并填写拆配表均价后，系统按 `拆配表总点数 × 拆配表均价 ÷ 原表总点数` 自动填写等值的原排表均价，从第六位小数开始四舍五入并固定显示五位，且不可编辑。
- 新单价 = 拆配表均价 + 最终调价，实肾按新单价计算，红补蓝退 = 原肾 − 实肾。
- 自动配平支持多角色联合求解和多组候选，Σ(拆配点数 × 调价) 必须严格等于 0。
- 每次进入第二步都必须先调整并确认完整热度顺序，不因点数、均价或角色集合一致而跳过；确认顺序是自动调价的唯一热度依据。
- 自动配平先锁定点数未变化角色，只调整点数有差异的角色；无精确解时依次加入中间热度角色、其他角色，最后才全面放宽。
- 仅 2～3 个角色变化且每个角色不超过 2 点时按小差异处理，二调幅度限制为原单价的 5%～10%；中差异为 10%～20%，大差异为 20%～40%。
- 热度前 25% 的角色优先限制大幅降价，后 25% 的角色优先限制大幅涨价；候选方案优先选择偏离原调价最小、修改角色最少的结果。
- 拆配表新增 1～2 个角色且全部位于热度前 25%（至少前两位）时，这些新增角色必须高于所有旧角色；两个新增角色之间继续按热度顺序拉开价差。该规则仅次于零误差和价格安全约束。
- “换调价”只保留方案间所有实际改动均在 0.5～10 元、且合计改动至少 1 元的组合；没有有效备选时按钮禁用。
- 自动方案仅根据热度提供参考，用户可继续手动核验和修改。

### v2.3 — 2026-07-01

- **二调自动配平**：以原排表调价（原单价 − 原均价）作为每个角色的初始值和求解锚点，在此基础上结合点数变化与用户确认热度进行二次调价；最终以整数分运算保证 `Σ(拆配点数×调价)=0`。
- **热度列排序**：二调配平器按“原排表单价 − 原排表均价”从高到低将角色整列从左到右排列，点数、变化、调价和新单价保持同列对应。
- **多组配平方案**：自动配平不再接受全零调价；可通过“换调价”循环查看多组严格零误差方案，并继续手动编辑。
- **分级调价差异约束**：自动配平以原调价为锚点。小差异二调幅度为原单价的 5%～10%，中差异为 10%～20%，大差异为 20%～40%；幅度约束作用于相对原调价的二次调整量。
- **分层放宽顺序**：第一层只修改点数差异角色；无精确解时加入中间热度角色，再加入其他角色，最后才在 40% 总上限内全面放宽。
- **热度两端保护**：热度前 25% 的角色优先限制大幅降价，后 25% 的角色优先限制大幅涨价；只有前序层无解时才逐步放宽。
- **多角色联合求解**：大额配平差值由多个角色共同承担，再进行精确收尾，避免双角色求解无解或将差额集中到单个角色。
- **角色热度确认**：每次导入两张排表进入第二步，都必须先调整并确认完整热度顺序；系统默认按原调价从高到低排列，确认后的从左到右顺序是所有角色自动调价的唯一热度依据。
- **新增角色前列规则**：拆配表新增 1～2 个角色且全部位于热度前 25%（至少前两位）时，自动方案强制这些新增角色高于所有旧角色，并保持新增角色之间的热度价差；优先级仅低于零误差和价格安全约束。
- **有效换调价**：备选方案与其他可切换方案之间，每个发生变化的角色调价差必须在 0.5～10 元，合计差至少 1 元；否则不计为有效方案。
- **拆配角色范围**：调价器以拆配表角色为准；原表存在但拆配表没有的角色按 0 点处理，不写入第二步调价表、确认表和新排表。
- **配平安全约束**：自动结果的新单价限制在 0 至拆配均价 3 倍之间；保留手动编辑与恢复自动结果功能。
- **暖色主题**：肾表、国际表和二调退补表新增琥珀、陶土、暖玫瑰、暖棕 4 套色盘，预览和 Excel 导出保持一致。

## FAQ

**Q: 为什么刷新后文件没了？**
A: 浏览器安全限制，File 对象无法序列化存储。刷新后草稿系统会恢复参数和主题，但需要重新上传文件。

**Q: 大批量文件会卡吗？**
A: 所有处理在浏览器主线程运行，建议单次不超过 50 个排表。国际表上限 50 个，肾表上限 10 个。

**Q: 支持什么 Excel 格式？**
A: 只支持 `.xlsx`。`.xls`、`.csv`、在线表导出须先用 Excel/WPS 另存为 `.xlsx`。

**Q: 数据安全吗？**
A: 纯前端，排表文件不会上传到任何服务器，所有计算在本地完成。
